# Roadmap (2026-06-20 snapshot)

## Summary

Leader 提供的版本規劃，涵蓋 v2.0.3（beta 待 release）、v2.1.0（Auto-Pause）、及未來方向。

## v2.0.3

所有功能已完成開發，等測試通過後 release stable。主要是 fleet command 平台統一和 instance warmup。

## v2.1.0 — Auto-Pause

核心設計：閒置 72hr → kill tmux（保留 session-id）→ 收到訊息時自動 wake（--resume）。

關鍵細節：
- `general` + `never_pause: true` 白名單豁免
- Paused 狀態用 marker file 持久化
- Wake 期間訊息排隊（message queue）
- `agend ls` 和 `/api/fleet` 顯示 ⏸ Paused

## 未來考慮

- Fleet /save /load（session 管理）
- Busy detection 改善（kiro thinking 偵測）
- registerBotCommands retry on timeout
- Web UI config editor
